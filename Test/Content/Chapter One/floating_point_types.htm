floating point types

Java provides two floating point types, float and double. They are called floating point because the decimal point “floats.” The decimal point is also referred to as the “radix.” Floating point numbers are a bit more complex than integers.

Confusingly, the number of digits available and the accuracy differ. It’s a kind of trade off between scale and precision. Float will hold up to four bytes of data, from -3.4E38 to +3.4E38. This is a shorthand way of writing 3.4 * 10^38. However, the accuracy is only to seven significant digits. In case that’s not confusing enough, floating point types will also hold a larger number of places closer to the origin than far from the origin. In other words, you can write small numbers to a higher number of decimal places than large ones. The smallest value that type float will hold is approximately -1.4E45.

Type double will hold eight bytes of data. The general specification for numbers of type double is -/+ 1.7E308; the smallest double is approximately -4.9E324. However, though the capacity to hold numbers is massive, the accuracy is only to 17 significant digits.

Most developers use int double as their default floating point type. The Java math library, which contains standard mathematical methods, uses double for most operations, and this avoids the need to convert between types. (insert cross reference) Type float is reserved for memory conservation purposes.

When type float is used, an f or F must be appended. Examples:

double pi = 3.14159;
float aFloat = 2.65f;
