character types

Character types are individual characters. It is important to distinguish them from type String, which is a basic and important type, but not a primitive. The String type is a group of characters, not individual characters. (insert cross reference) When Java was originally released, it used the ASCII character set, based upon the English language, which includes support for only 128 characters. Current releases of Java instead use Unicode, which provides over 110,000 characters, including support for Cyrillic, Arabic, phonetic Japanese, and other alphabets. Support is also available for a wide variety of symbols, from currency symbols to mathematical operators.

When we deal with characters, we are generally concerned with String types. Char types, because they are primitives, are more economically processed than String types. Many String methods operate on char types. These methods are too numerous to elaborate in their entirety here. Among the more important char type operations on strings are; decomposition; ordering; identifying; changing case; and changing position. We will deal more extensively with these operations when we discuss String types. (insert cross reference)

Characters are also used in escape sequences. Escape sequences are reserved for characters which cannot be entered by default from the keyboard. Many of these are control characters, which control the output of text by inserting a tab, a new line, etc. For example, the escape sequence for a new line is \n. (insert table of control characters) Printing to the console, or monitor, makes extensive use of control characters to control the appearance of output. Another use for escape sequences is to enter a symbol or other special character. Thus, to insert the symbol for Sterling, £,type \u00A3, which is the hexadecimal value for the symbol in Unicode. Escape sequences are always preceded by the backslash character, \, the letter u, and generally employ Unicode, though other encoding schemes are recognized.
